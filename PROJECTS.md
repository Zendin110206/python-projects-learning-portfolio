## Per-Project Completion Standard

Each completed project should include:

- A concise project README.
- Clear input and output behavior.
- Source code that can be run from a documented command.
- Tests when the project has deterministic logic.
- Notes about limitations and possible improvements.
- A commit that represents a real checkpoint, not a noisy micro-change.
